# 阶段 2: 判断共识 - Orchestrator

## 概述

分析 Opus 和 Codex 的审查结果，判断是否达成共识。

## 任务

分析阶段 1 收到的两条审查结果消息，判断结论是否一致。

## 关注点

- 是否发现问题？
- 发现了哪些问题？
- 问题的严重程度如何？
- 两者的发现是否有交集？

## 判断结果

| 结果          | 条件                    | 下一阶段             |
| ------------- | ----------------------- | -------------------- |
| `both_ok`     | 双方都没发现问题        | → 阶段 5（汇总）     |
| `same_issues` | 双方发现相同/相似的问题 | → 阶段 4（修复）     |
| `divergent`   | 结论有分歧              | → 阶段 3（交叉确认） |

## 执行

```bash
duo-cli set stage 2

# 分析后记录结果
duo-cli set s2:result both_ok      # 或 same_issues / divergent
```

然后进入对应阶段。
